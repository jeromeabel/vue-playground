# Vue Playground Reset вЂ” Design Spec

## Overview

Reset the `vue-playground` repository from a Vue shop demo into a botanical data explorer that serves as a playground/showcase for technical blog posts. The app uses the GBIF Species API and a pre-fetched local dataset to demonstrate TanStack Query caching, virtualization performance comparisons, and SPA chunk error handling.

## Data Layer

### Live API вЂ” GBIF Species API

Used by `/species` routes for the TanStack Query caching blog post. Base URL: `https://api.gbif.org`.

No authentication required. CC0 license for metadata. GBIF has no published hard rate limit for anonymous users, but the fetch script should throttle requests (~100ms between calls) to be a good API citizen.

#### Endpoints and Response Types

**`GET /v1/species/search`** вЂ” paginated species list

Query params: `rank=SPECIES`, `highertaxon_key=6` (Plantae kingdom), `status=ACCEPTED`, `limit=20`, `offset=0`

Fields consumed by the app:
```ts
interface GbifSearchResponse {
  offset: number
  limit: number
  endOfRecords: boolean
  count: number // total matching species вЂ” used for pagination UI
  results: GbifSpeciesSummary[]
}

interface GbifSpeciesSummary {
  key: number
  scientificName: string
  canonicalName: string
  kingdom: string
  phylum: string
  class: string
  order: string
  family: string
  genus: string
  taxonomicStatus: string
  rank: string
}
```

**`GET /v1/species/{key}`** вЂ” species detail (same fields as `GbifSpeciesSummary` plus additional text fields)

Additional fields consumed:
```ts
interface GbifSpeciesDetail extends GbifSpeciesSummary {
  authorship: string
  publishedIn?: string
  nameType: string
  numDescendants: number
}
```

**`GET /v1/species/{key}/media`** вЂ” images

Fields consumed:
```ts
interface GbifMediaResponse {
  results: Array<{
    type: string // "StillImage"
    identifier: string // image URL
    title?: string
    creator?: string
    license?: string
  }>
}
```

**`GET /v1/species/{key}/vernacularNames`** вЂ” common names

Fields consumed:
```ts
interface GbifVernacularNamesResponse {
  results: Array<{
    vernacularName: string
    language: string // "fra", "eng", etc.
    source?: string
  }>
}
```

### Local JSON вЂ” Pre-fetched Dataset

Used by `/benchmark` routes for virtualization comparisons.

- File: `public/data/species-3000.json`
- Generated by: `scripts/fetch-species.ts`
- The script fetches 3 pages of 1000 items from GBIF (`limit=1000`, `offset=0/1000/2000`) with `~100ms` delay between requests
- ~3000 items using the `GbifSpeciesSummary` type (taxonomic fields only вЂ” `habitat`, `threatStatus`, and `numOccurrences` are NOT available from the Species Search API and are excluded)
- Deterministic, works offline, reproducible benchmarks

## Feature 1: TanStack Query Caching (Blog Post)

**Routes**: `/species` (list) + `/species/:id` (detail)

### Patterns Demonstrated

- **Paginated queries** with `placeholderData: keepPreviousData` (TanStack Query v5 API) вЂ” smooth page transitions without flash
- **`staleTime`** вЂ” 5-minute staleTime avoids refetches on back-navigation
- **`initialData`** вЂ” pre-populate detail page from list cache for instant navigation
- **Prefetching** вЂ” hover on a species card в†’ prefetch detail + media
- **Dependent queries** вЂ” detail page fetches species, then media, then vernacular names in sequence
- **DevTools** вЂ” `@tanstack/vue-query-devtools` included for visual cache inspection

### Components

- `SpeciesCard.vue` вЂ” card in the paginated list, triggers prefetch on hover
- `pages/species/index.vue` вЂ” paginated species list with pagination controls
- `pages/species/[id].vue` вЂ” detail page with dependent queries for media and names

### Composables

- `useSpeciesList(page)` вЂ” paginated query with `placeholderData: keepPreviousData` and prefetching
- `useSpeciesDetail(id)` вЂ” detail query with `initialData` from list cache
- `useSpeciesMedia(id)` вЂ” dependent query, enabled only when detail is loaded
- `useVernacularNames(id)` вЂ” dependent query for common names

## Feature 2: Virtualization Benchmark (Blog Post)

**Routes**: `/benchmark/*` вЂ” three sub-pages rendering the same 3000-item dataset.

### Pages

| Page | Approach | Purpose |
|------|----------|---------|
| `/benchmark/basic-table` | Plain `v-for` `<table>` | Baseline вЂ” shows how 3000 rows degrade DOM performance |
| `/benchmark/primevue-table` | PrimeVue `<DataTable>` with virtual scroll | Popular component library approach |
| `/benchmark/tanstack-table` | TanStack Table + TanStack Virtual | Headless + virtualized approach |

### Metrics Panel

Each benchmark page includes a `MetricsPanel.vue` component displaying:
- DOM node count (via `MutationObserver`, updated on mutations)
- Initial mount time (measured with `performance.now()` between `onBeforeMount` and `onMounted`)
- FPS counter (via `requestAnimationFrame` loop, displayed as rolling average)

### Performance Analysis Scripts

Located in `scripts/analyze/`:
- `analyze_lighthouse.py` вЂ” parses Lighthouse JSON reports, generates comparison charts (matplotlib)
- `analyze_trace.py` вЂ” parses Chrome Performance traces, extracts scripting/rendering/painting time, generates per-second normalized graphs
- `requirements.txt` вЂ” Python dependencies (`matplotlib`, `seaborn`, `json5`). Run with `python -m venv .venv && pip install -r scripts/analyze/requirements.txt`

### PrimeVue Isolation

PrimeVue is imported only in the `/benchmark/primevue-table` route via dynamic import. It does not affect the rest of the app's bundle.

## Feature 3: Chunk Error Handling (Blog Post)

**Scope**: Cross-cutting вЂ” applies to all lazy routes and async components.

### Implementation вЂ” `src/utils/chunk-reload.ts`

1. **`vite:preloadError` listener** вЂ” single listener catches all chunk failures (routes and components)
2. **Navigation bridge** вЂ” plain object (not reactive) that router guards write to and the error listener reads from, to determine if a navigation is pending
3. **`sessionStorage` reload guard** вЂ” prevents infinite reload loops for genuinely broken chunks
4. **`event.preventDefault()`** вЂ” prevents Vite from re-throwing the error when a reload is in progress

### Version Banner вЂ” `src/components/VersionBanner.vue`

- Polls `/version` file every 60 seconds (disabled in dev mode via `import.meta.env.DEV` check)
- Non-blocking banner (no overlay, no forced refresh)
- Dismissible (in-memory, resets on manual refresh)
- `/version` file generated at build time by `scripts/generate-version.ts` using Netlify's `DEPLOY_ID` or `COMMIT_REF`, falling back to `git rev-parse --short HEAD`, falling back to `"dev"`

### File-Based Routing Compatibility

The navigation bridge hooks into the router instance via `router.beforeEach` / `router.afterEach`. File-based routing generates the route definitions, but the router instance and its guards are unaffected вЂ” no compatibility issue.

### Tests

Unit tests (Vitest) covering 4 key behaviors:
1. No pending navigation в†’ `window.location.reload()`
2. Pending navigation в†’ `window.location.assign(href)`
3. Guard prevents infinite reload (same path already in `sessionStorage`)
4. Resilient when `sessionStorage` throws (private browsing)

Plus 2 guard lifecycle tests:
5. `clearIfNavigatedAway` resets guard on different path
6. `clearIfNavigatedAway` preserves guard on same path

## Pages & Navigation

### `AppHeader.vue`

Simple top navigation bar with links to: **Species** (`/species`), **Benchmark** (`/benchmark`), **About** (`/about`). Active route is highlighted. Logo/title "Vue Playground" links to `/`.

### `pages/index.vue` вЂ” Home

Landing page with a brief description of the playground and cards linking to the three main sections (Species Explorer, Benchmark, About). Each card shows the section name and a one-line description.

### `pages/species/index.vue` вЂ” Species List

Paginated grid of `SpeciesCard` components. Pagination controls: **Previous / Next** buttons + current page indicator + total count from API. Page size: 20 items (matching API `limit=20`).

### `pages/species/[id].vue` вЂ” Species Detail

Species detail with: scientific name, taxonomy breadcrumb (kingdom в†’ genus), authorship, images (from media endpoint), French/English common names. Loading skeleton while dependent queries resolve. Error state with retry button if API fails.

### `pages/benchmark/index.vue` вЂ” Benchmark Hub

Navigation page with cards linking to the three benchmark sub-pages. Each card shows the approach name and a short description.

### `pages/about.vue` вЂ” About

Brief page explaining the playground's purpose: a showcase for technical blog posts, with links to the blog posts (external) and the GitHub repository.

### Error States

All pages using live API calls show: skeleton loaders during loading (via TanStack Query's `isPending`), and an error message with retry button on failure (via `isError` + `refetch`).

- `public/version` generated during build via Netlify's `DEPLOY_ID` env var
- Immutable caching headers for hashed assets (`Cache-Control: max-age=31536000, immutable`)

## Reset Procedure

### Step 1: Backup

1. Export all 11 GitHub issues to `backup/issues/` as JSON (via `gh issue list --json` + `gh issue view --json`)
2. Copy `.github/ISSUE_TEMPLATE/user-story.md` to `backup/templates/`
3. `draft-for-blogs/` is preserved in git history вЂ” no explicit backup needed

### Step 2: Clean Slate

Git history is preserved (no force-push). The reset is a single commit that:
1. Deletes all files under `src/`, `vite.config.ts`, `tsconfig*.json`, `index.html`, `package.json`, `package-lock.json`, `tailwind.config.*`, `postcss.config.*`, `.eslintrc.*`, `.prettierrc.*`
2. Removes `node_modules/`
3. Scaffolds the new project from scratch using `pnpm create vue@latest` (or manual setup) with the specified stack
4. Replaces `package-lock.json` with `pnpm-lock.yaml`
5. Keeps `draft-for-blogs/`, `backup/`, `.github/`, `README.md`, and `docs/` intact

## Out of Scope

- Authentication / user accounts
- Database / persistence beyond TanStack Query cache
- SSR / Nuxt
- E2E testing (Playwright/Cypress) вЂ” may be added later
- i18n вЂ” GBIF provides French vernacular names, but the UI is in English
